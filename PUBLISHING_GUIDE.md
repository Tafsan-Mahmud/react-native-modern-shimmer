# Publishing Guide — react-native-modern-shimmer

Step by step from zero to published on npm and GitHub.

---

## Step 1 — Create npm account

1. Go to https://www.npmjs.com/signup
2. Create an account (remember your username — it goes in package.json)
3. Verify your email

---

## Step 2 — Create GitHub repository

1. Go to https://github.com/new
2. Repository name: `react-native-modern-shimmer`
3. Set to Public
4. Do NOT add README (we already have one)
5. Click "Create repository"

---

## Step 3 — Update package.json

Open `package.json` and replace:
- `YOUR_USERNAME` → your GitHub username
- `Your Name` → your real name
- `your@email.com` → your email

---

## Step 4 — Push code to GitHub

Run these commands in the package folder:

```bash
git init
git add .
git commit -m "initial release"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/react-native-modern-shimmer.git
git push -u origin main
```

---

## Step 5 — Build the package

Install dependencies and build:

```bash
npm install
npm run build
```

This generates the `lib/` folder with CommonJS, ES module, and TypeScript declaration files.

---

## Step 6 — Login to npm

```bash
npm login
```

Enter your npm username, password, and email when prompted.

---

## Step 7 — Publish manually (first time)

```bash
npm publish --access public
```

Your package is now live at:
`https://www.npmjs.com/package/react-native-modern-shimmer`

---

## Step 8 — Set up auto-publish with GitHub Actions (optional but recommended)

This publishes automatically every time you push a version tag.

1. Go to https://www.npmjs.com → Account → Access Tokens
2. Click "Generate New Token" → Select "Automation"
3. Copy the token

4. Go to your GitHub repo → Settings → Secrets and variables → Actions
5. Click "New repository secret"
6. Name: `NPM_TOKEN`
7. Value: paste the token you copied
8. Click "Add secret"

Now to publish a new version:

```bash
# Bump version in package.json (e.g. 1.0.0 → 1.0.1)
npm version patch   # or minor or major

# Push the tag — GitHub Actions publishes automatically
git push --follow-tags
```

---

## Versioning guide

| Command | When to use | Example |
|---|---|---|
| `npm version patch` | Bug fixes | 1.0.0 → 1.0.1 |
| `npm version minor` | New features, backwards compatible | 1.0.0 → 1.1.0 |
| `npm version major` | Breaking changes | 1.0.0 → 2.0.0 |

---

## After publishing

1. Add topics to your GitHub repo: `react-native`, `expo`, `shimmer`, `skeleton`, `loading`, `animation`
2. Add a demo GIF to the README (record with iOS/Android simulator)
3. Share on Reddit r/reactnative and Twitter/X for visibility

## File structure explained

```
react-native-modern-shimmer/
├── src/
│   ├── Shimmer.tsx      ← The component
│   └── index.ts         ← Public exports
├── lib/                 ← Generated by build (don't edit)
│   ├── commonjs/        ← For Node/Metro
│   ├── module/          ← For bundlers (ESM)
│   └── typescript/      ← Type declarations
├── .github/
│   └── workflows/
│       └── publish.yml  ← Auto-publish on git tag
├── package.json
├── tsconfig.json
├── LICENSE
├── README.md
└── .npmignore
```
